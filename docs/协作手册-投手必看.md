# 多站群多 agent · 协作手册

**给:** ZJB / HZM / CHJ / HNN(投手)+ 承接层 + 技术

**3 个核心动作**

| 你想做什么 | 怎么做 | 多久看到结果 |
|---|---|---|
| 看自己组数据 | 浏览器打开 GitHub Pages 链接 | 立即 |
| **上传 FB 后台 xlsx 自动 ingest** | **拖文件到 `requests/uploads/`** | **5-7 分钟** |
| 触发 agent 跑 | 在 GitHub `requests/` 网页编辑创建文件 | 5-10 分钟 |
| 改自己打法包 | 在 GitHub 网页编辑 `打法包-投手能力/投手{你}-*.yaml` | 5-10 分钟 |

## 1. 看板 URL

```
https://{ZJB github 账号}.github.io/multisite-platform/
```

主页是总览(ZJB 看)。点上方 nav 切换到自己投手的页面:
- `HZM.html` - HZM 自看
- `CHJ.html` - CHJ 自看
- `HNN.html` - HNN 自看

**你只看到自己的组,看不到别人的(权限隔离)。**

## 2. 触发 agent 的标准动作(以 HZM 为例)

### 网页操作 5 步
1. 浏览器打开 `https://github.com/{ZJB账号}/multisite-platform/tree/main/requests`
2. 右上角 `Add file` → `Create new file`
3. 文件名输入框填:`HZM-数据-20260513-1530.md`(`HZM-{描述}-{时间}.md`)
4. 文件内容(直接 copy):

```
---
agent: 数据分析
owner: HZM
---
说明你的诉求(可空)
```

5. 滚到页面底 → 绿色按钮 `Commit new file`

### 等 5-10 分钟
ZJB 这边 cron 每 1 分钟扫一次,看到你的请求 → 跑 agent → 推回 GitHub → 你刷新看板就有新数据。

### 怎么知道处理完了
1. 你的请求文件会自动从 `requests/` 移到 `requests/_done/`
2. 看板上对应你那一栏会出现新一条 agent 输出

## 3. 改自己打法包(改阈值)

**前提:你只能改自己的打法包,别人的不许动。**

### 操作步骤
1. 浏览器打开 `https://github.com/{ZJB账号}/multisite-platform/blob/main/platform-core/能力包/打法包-投手能力/投手HZM-成人付费-激进.yaml`
2. 右上角铅笔图标 `Edit this file`
3. 改你的阈值(看代码注释,改数字就行)
4. 滚到底,Commit message 写"改 XX 阈值为 YY",绿色按钮 commit

**改完不需要触发请求**,ZJB cron 下次跑 agent 时自动用新阈值。

## 4. 11 个 agent 各干什么

| Agent 名 | 干什么 | 触发时填 agent: |
|---|---|---|
| 数据分析 | 看自己组的花费/HVU/CPHQ + 灯色 | `数据分析` |
| 归因诊断 | ROI 异动时排 5 种外因 | `归因诊断` |
| 知识沉淀 | 数据→可证伪命题(三要素校验) | `知识沉淀` |
| 策略简报 | 可信命题→投放方向(Hook/KPI) | `策略简报` |
| 文案 | Hook/正文/评论区(反 AI 腔) | `文案` |
| 故事脚本 | 视频脚本(4 镜头 + prompt) | `故事脚本` |
| 素材审核 | 禁用词/调性/反 AI 腔检查 | `素材审核` |
| 研发需求 | UX 缺口→P0/P1/P2 技术需求 | `研发需求` |
| 意志继承 | S 类杠杆三拆/B 类零 HVU 诊断 | `意志继承` |
| 搬运 | 子站→总站数据回灌 | `搬运` |
| 交接消息 | 策略→飞书/微信粘贴版 | `交接消息` |

## 5. 看到异常怎么办

| 看到 | 含义 | 你做什么 |
|---|---|---|
| `status: failed` | agent 跑失败 | 看 `gap_reason` 字段,告诉 ZJB |
| `status: no_data` | 没数据可分析 | 检查 ingest 有没有推数 |
| 请求文件 24h 没进 _done | ZJB 那边没扫到 | 微信问 ZJB |
| 看板数据日期不对 | 数据没更新 | 等 daily_close 跑(默认 14:00) |

## 6. FAQ · 第一次用蒙了看这里

### Q1 · 我点了"跑数据分析"按钮,跳到 GitHub,然后呢?

GitHub 会打开"创建新文件"页面,**文件名和内容都已自动填好**,你只需要:
1. 滚到页面**最底**
2. 找到绿色按钮 **"Commit new file"**(可能在右下)
3. 点它

完成。这就是你的请求,ZJB 那边会在 60 秒内开始处理。

### Q2 · 找不到 "Commit new file" 按钮?

页面太长滚到底,绿色按钮一定在最下面。如果看到的是"Propose new file"也没事,功能一样,点就行。

### Q3 · 我要等多久?

- 60 秒内:ZJB 那边 monitor 扫到你的请求
- 1-2 分钟:agent 跑完
- 5-10 分钟:看板自动更新(GitHub Pages 重渲染慢)

5 分钟后回看板刷新就能看到。

### Q4 · 怎么知道我的请求处理完了?

3 种确认方式:
1. **去 requests/ 目录**看你刚才提的文件还在不在
   - 在 → 还在处理
   - 不在(自动消失了)→ 已处理,跑去 `_done/` 看归档结果
2. **看板上**:总览页"最近请求"区块会显示你这条已处理
3. **看自己的 view 页**(HZM.html),最新一条 agent 输出就是你刚刚触发的

### Q5 · 我提的请求要带什么参数吗?

不用。点按钮跳过去后,frontmatter 已经填好:
```
agent: 数据分析
owner: HZM
```
你想说点什么直接在下面写正文就行(可空)。

### Q6 · 我想改自己的打法包阈值,在哪改?

**不在 requests/ 里改**,直接编辑你的打法包 yaml:

`platform-core/能力包/打法包-投手能力/投手HZM-成人付费-激进.yaml`

GitHub 网页打开 → 右上角铅笔图标 Edit → 改数字 → Commit changes。改完不需要提请求,下次自动用新阈值。

### Q7 · 我怎么知道哪个 agent 该用?

打开你的 view 页(HZM.html),顶部有 11 个按钮,每个按钮**上面有用途说明**(hover 还会有 tooltip)。常用 4 个标蓝色(数据分析/归因诊断/知识沉淀/策略简报),其它灰色。

### Q8 · 我能提多个请求一起跑吗?

可以。每点一次按钮就提一个新文件,monitor 会按时间顺序逐个处理。

### Q9 · 我请求的 agent 失败了怎么办?

去 `requests/_done/` 找你的文件,文件末尾会写处理结果。如果 status=failed,把 gap_reason 截图发 ZJB,他知道怎么修。

### Q10 · 我能看 CHJ / HNN 的数据吗?

不能直接看,**视角是隔离的**。你只能看自己组(HZM.html)。如果有跨投手的需求,跟 ZJB 说让他用 ZJB 视角看。

## ZJB 视角

### 第一次配置(只做一次)

```bash
cd ~/Documents/GitHub/multisite-platform

# 1. 配 GitHub remote
git remote add origin git@github.com:你的账号/multisite-platform.git
git push -u origin main

# 2. GitHub repo → Settings → Pages
#    Source = GitHub Actions(workflow 自动部署)
#    1-2 分钟生成 URL: https://你的账号.github.io/multisite-platform/

# 3. GitHub repo → Settings → Actions → General
#    Workflow permissions = Read and write permissions
#    保存
```

### 全自动化(不需要本地服务)

**P0-9.14 起所有处理跑在 GitHub Actions 云端**:
- HZM 提请求 / 上传 xlsx / 改阈值 → push GitHub
- workflow 自动触发 → 跑 ingest + agent + 重建 dashboard + push 回
- ZJB Mac **完全不需要开机**,**也不需要跑 monitor**

### 本地手动跑(可选,调试用)

```bash
cd ~/Documents/GitHub/multisite-platform

# 一次性跑(类似 Actions workflow 的本地等价)
bash bin/start.sh --ingest-only

# 持续监听(老版本,P0-9.14 后建议不用,Actions 已替代)
bash bin/start.sh
bash bin/start.sh --stop
```

### 看 Actions 跑的状态

打开 https://github.com/zhujiebing2020-lgtm/multisite-platform/actions
看到每次请求触发的 workflow run · 绿 ✓=成功 / 红 ✗=失败

### 看 log

```bash
tail -f /tmp/multisite-git-sync.log     # 本地 git sync(老版)
tail -f /tmp/multisite.log              # 本地 monitor(老版)
```
Actions 的 log 在 GitHub 网页 Actions tab 里看(每次 run 点进去)。
