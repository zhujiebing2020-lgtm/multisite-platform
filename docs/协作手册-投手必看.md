# 多站群多 agent · 协作手册

**给:** ZJB / HZM / CHJ / HNN(投手)+ 承接层 + 技术

**3 个核心动作**

| 你想做什么 | 怎么做 | 多久看到结果 |
|---|---|---|
| 看自己组数据 | 浏览器打开 GitHub Pages 链接 | 立即 |
| 触发 agent 跑 | 在 GitHub `requests/` 网页编辑创建文件 | 5-10 分钟 |
| 改自己打法包 | 在 GitHub 网页编辑 `打法包-投手能力/投手{你}-*.yaml` | 5-10 分钟 |


## 2. 触发 agent 的标准动作(以 HZM 为例)

### 网页操作 5 步
1. 浏览器打开 `https://github.com/{ZJB账号}/multisite-platform/tree/main/requests`
2. 右上角 `Add file` → `Create new file`
3. 文件名输入框填:`HZM-数据-20260513-1530.md`(`HZM-{描述}-{时间}.md`)
4. 文件内容(直接 copy):

```
---
agent: 数据分析
owner: HZM
---
说明你的诉求(可空)
```

5. 滚到页面底 → 绿色按钮 `Commit new file`

### 等 5-10 分钟
ZJB 这边 cron 每 1 分钟扫一次,看到你的请求 → 跑 agent → 推回 GitHub → 你刷新看板就有新数据。

### 怎么知道处理完了
1. 你的请求文件会自动从 `requests/` 移到 `requests/_done/`
2. 看板上对应你那一栏会出现新一条 agent 输出

## 3. 改自己打法包(改阈值)

**前提:你只能改自己的打法包,别人的不许动。**

### 操作步骤
1. 浏览器打开 `https://github.com/{ZJB账号}/multisite-platform/blob/main/platform-core/能力包/打法包-投手能力/投手HZM-成人付费-激进.yaml`
2. 右上角铅笔图标 `Edit this file`
3. 改你的阈值(看代码注释,改数字就行)
4. 滚到底,Commit message 写"改 XX 阈值为 YY",绿色按钮 commit

**改完不需要触发请求**,ZJB cron 下次跑 agent 时自动用新阈值。

## 4. 11 个 agent 各干什么

| Agent 名 | 干什么 | 触发时填 agent: |
|---|---|---|
| 数据分析 | 看自己组的花费/HVU/CPHQ + 灯色 | `数据分析` |
| 归因诊断 | ROI 异动时排 5 种外因 | `归因诊断` |
| 知识沉淀 | 数据→可证伪命题(三要素校验) | `知识沉淀` |
| 策略简报 | 可信命题→投放方向(Hook/KPI) | `策略简报` |
| 文案 | Hook/正文/评论区(反 AI 腔) | `文案` |
| 故事脚本 | 视频脚本(4 镜头 + prompt) | `故事脚本` |
| 素材审核 | 禁用词/调性/反 AI 腔检查 | `素材审核` |
| 研发需求 | UX 缺口→P0/P1/P2 技术需求 | `研发需求` |
| 意志继承 | S 类杠杆三拆/B 类零 HVU 诊断 | `意志继承` |
| 搬运 | 子站→总站数据回灌 | `搬运` |
| 交接消息 | 策略→飞书/微信粘贴版 | `交接消息` |

## 5. 看到异常怎么办

| 看到 | 含义 | 你做什么 |
|---|---|---|
| `status: failed` | agent 跑失败 | 看 `gap_reason` 字段,告诉 ZJB |
| `status: no_data` | 没数据可分析 | 检查 ingest 有没有推数 |
| 请求文件 24h 没进 _done | ZJB 那边没扫到 | 微信问 ZJB |
| 看板数据日期不对 | 数据没更新 | 等 daily_close 跑(默认 14:00) |

## ZJB 视角

### 第一次配置(只做一次)

```bash
cd ~/Documents/GitHub/multisite-platform

# 1. 配 GitHub remote
git remote add origin git@github.com:你的账号/multisite-platform.git
git push -u origin main

# 2. GitHub repo → Settings → Pages
#    Source = main 分支 / Path = /view
#    1-2 分钟生成 URL: https://你的账号.github.io/multisite-platform/

# 3. 把 URL 发给 HZM
```

### 日常启动(起一个后台监听器)

```bash
# 方式 A · 手动起(关终端就停)
cd ~/Documents/GitHub/multisite-platform
python3 bin/request_monitor.py --watch

# 方式 B · 后台起(关终端继续跑)
cd ~/Documents/GitHub/multisite-platform
nohup python3 bin/request_monitor.py --watch > /tmp/multisite.log 2>&1 &

# 方式 C · cron 定时扫(单次扫描模式)
crontab -e
# 加一行:
*/1 * * * * cd /Users/zhujiebing/Documents/GitHub/multisite-platform && python3 bin/request_monitor.py >> /tmp/multisite-cron.log 2>&1
```

### 手动跑一次(调试)

```bash
python3 bin/ingest_daily.py --dates 2026-05-12 2026-05-13   # 真 ingest 两天
python3 bin/export_dashboard.py                              # sqlite → cards json
python3 bin/build.py                                         # cards → HTML
bash bin/git_sync.sh                                         # 自动 commit + push
```

### 自动 sync 逻辑

`request_monitor.py` 处理完请求 → 自动调 `git_sync.sh`:
- 只 add `data/cards/` + `view/` + `requests/_done/`(不 add 代码改动)
- 没 remote 时静默跳过(本地调试不会报错)
- commit + push 失败不崩,下次重试

### 看 log

```bash
tail -f /tmp/multisite-git-sync.log      # git sync 日志
tail -f /tmp/multisite.log               # monitor 日志(方式 B)
tail -f /tmp/multisite-cron.log          # cron 日志(方式 C)
```
